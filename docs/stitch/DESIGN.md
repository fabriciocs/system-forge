# DESIGN

## Direção

Aplicar a linguagem `Editorial Control Plane` com a identidade `Layered Node Frame`.

## Objetivo

Padronizar a experiência visual para todas as telas do produto sem gerar código final.

## Regras

- Usar composição em camadas.
- Preferir blocos densos e bem alinhados.
- Explicitar estado, permissão e contexto.
- Evitar UI genérica de SaaS.
- Respeitar tokens, contraste e foco.

## Componentes globais

- Botão
- Campo
- Card
- Tabela
- Modal
- Drawer
- Banner
- Toast
- Stepper
- Diff viewer

## Qualidade

- Compatível com Material Design 3.
- Compatível com WCAG 2.2.
- Funciona em mobile, tablet e desktop.
